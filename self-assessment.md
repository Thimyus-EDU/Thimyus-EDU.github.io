# Professional Self-Assessment

Completing the Computer Science program and developing my CS 499 ePortfolio has helped me better understand my growth as a software developer and my professional direction within the field of computer science. Throughout the program, I built skills in software design, programming, testing, algorithms and data structures, databases, security, and technical communication. The capstone gave me an opportunity to bring those skills together in one focused project by taking an earlier academic artifact and enhancing it into a more complete, maintainable, and practical software application.

My selected artifact is a task management application that originally began as a Java project from earlier coursework. The original version demonstrated basic object-oriented programming, validation, and service-layer logic, but it was limited in scope because it did not provide a full user-facing workflow, advanced task organization, or database-backed persistence. For the capstone, I enhanced the artifact across three categories: software design and engineering, algorithms and data structures, and databases. These enhancements allowed me to show how a simple academic project can be expanded into a more professional application through modular design, improved logic, stronger data handling, and secure database practices.

One of the most important strengths I developed throughout the program is the ability to design software with maintainability in mind. Earlier in my coursework, many assignments focused on meeting specific functional requirements. As I progressed, I began to understand that good software is not only measured by whether it works, but also by how well it can be understood, tested, modified, and expanded. In the capstone project, I applied this lesson by separating the task management application into clear components. The parser handles user commands, the task service manages business logic, the repository manages data persistence, and the task model defines the structure of the data. This separation of concerns helped turn the project into a more organized application and demonstrated my ability to make design decisions that support future growth.

The ePortfolio also helped me showcase my ability to evaluate trade-offs in software design. For example, the project originally used simple in-memory storage, then JSON-based persistence, and finally SQLite database storage. Each stage had advantages and limitations. In-memory storage was simple but temporary. JSON persistence allowed data to remain between runs, but it treated the task list as one file-based object. SQLite introduced a more realistic database structure, allowing tasks to be stored as records and retrieved through SQL queries. This progression helped me better understand how implementation choices affect scalability, maintainability, and reliability.

The program also strengthened my understanding of algorithms and data structures. In the task management enhancement, I applied searching, filtering, sorting, and priority-based retrieval to make the application more useful. Searching and filtering required the program to examine task data and return only the records that matched the user’s needs. Sorting by priority and due date improved the way users could organize work. The highest-priority task feature demonstrated how a priority-based structure can help identify the next task that should receive attention. These features showed me how algorithms are not only abstract concepts from coursework; they directly affect how useful and efficient an application feels to the user.

My coursework outside of the capstone also contributed to this growth. Through assignments involving data structures, object-oriented programming, testing, and reverse engineering, I practiced breaking complex problems into smaller parts and reasoning through how software behaves. Testing-focused coursework helped me think more carefully about validation, edge cases, and expected behavior. Reverse engineering and low-level analysis helped me appreciate how software works beneath the surface and why precise logic matters. These experiences helped prepare me to evaluate code more carefully during the capstone and to explain both what the program does and why certain changes improve it.

Communication has also been an important part of my development. In computer science, technical skill is not enough if ideas cannot be explained clearly to other developers, instructors, stakeholders, or users. The code review portion of CS 499 gave me practice communicating technical decisions in a professional way. Instead of only describing what the code did, I had to explain the existing functionality, identify areas for improvement, and justify planned enhancements. The written narratives strengthened this skill further because each enhancement required me to explain why the artifact was selected, how it was improved, what challenges I faced, and which course outcomes were demonstrated. This helped me practice adapting technical communication to an audience that needs both enough detail to understand the work and enough context to understand its value.

Collaboration and code review are also important professional skills that I developed through the program. Even when working individually, I learned to approach software development as if another person may need to read, review, maintain, or build on my code later. This influenced how I organized the capstone project and how I explained my design choices. Clear naming, modular files, focused responsibilities, and readable control flow all support collaboration because they make it easier for another developer or stakeholder to understand the project. Code review also supports better decision making because it creates a structured opportunity to identify weaknesses, discuss alternatives, and improve the final product before deployment.

The capstone also helped me strengthen my security mindset. Security is not only a concern for large web applications or enterprise systems; it begins with everyday development habits. In my task management application, I applied input validation, defensive handling of invalid commands, and prepared statements for database operations. Using prepared statements in the SQLite enhancement helped reduce the risk of unsafe SQL construction and reinforced the importance of treating user input carefully. This experience helped me understand that secure design should be part of the development process from the beginning rather than something added only after a system is complete.

My work with databases was another important area of growth. Adding SQLite persistence required me to think about how application data should be structured, stored, retrieved, updated, and deleted. The database enhancement required CRUD operations, a tasks table, SQL queries, and a repository layer that kept database logic separate from command parsing and business logic. This helped me connect software engineering and database design because the structure of the database influenced how the rest of the application interacted with task records. It also helped me better understand the value of persistence and structured data management in real applications.

Developing the ePortfolio has also helped shape my professional goals. My goal is to continue growing as a software engineer who can design practical, maintainable, and secure applications. The capstone project reflects that goal because it demonstrates a complete development path: analyzing an existing artifact, identifying weaknesses, planning improvements, implementing enhancements, testing functionality, and explaining the results. The final application is still small in scope, but the process used to improve it reflects professional software development practices that can scale to larger systems.

The artifacts in this portfolio fit together as a single story of growth. The first enhancement focuses on software design and engineering by transforming the original task project into a modular C++ command-line application. The second enhancement focuses on algorithms and data structures by adding search, filter, sort, and priority-based task retrieval features. The third enhancement focuses on databases by replacing file-based persistence with SQLite database storage and prepared statements. Together, these enhancements show how I can improve an application across multiple areas of computer science while maintaining a cohesive design.

Overall, my CS 499 ePortfolio represents my progress from completing individual programming assignments to thinking more like a software engineer. I have learned to evaluate existing code, identify opportunities for improvement, make design trade-offs, apply algorithms and data structures, integrate database storage, communicate technical decisions, and consider security throughout development. These skills prepare me to contribute to software projects in a professional environment and continue developing as a thoughtful, adaptable, and security-minded computer science graduate.
